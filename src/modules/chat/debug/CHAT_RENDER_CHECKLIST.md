# Чеклист: проверка ререндеров в чате

Включи логи: в `chat-render-log.ts` установи `ENABLE_CHAT_RENDER_LOGS = true`.  
Запусти приложение, открой чат и выполняй действия по порядку. После **каждого** пункта скинь логи (все строки `[ChatRender] ...` с момента действия до стабилизации).

Формат логов: `[ChatRender] <Type> [details] #<count>`
- **Chat** — ререндер экрана Chat
- **MessagesList** — ререндер списка сообщений
- **MessageItem** — ререндер одной ячейки сообщения (msgId=...)
- **renderItem** — вызов renderItem FlatList’ом (создание/обновление ячейки)

---

## 1. Открытие чата
- **Действие:** Зайти в любой чат с сообщениями, не скроллить.
- **Ожидание:** Несколько Chat/MessagesList при монтировании, затем renderItem только для видимых сообщений (initialNumToRender).
- **Пришли:** Все `[ChatRender]` с момента открытия экрана чата.

---

## 2. Скролл вверх/вниз
- **Действие:** Плавно проскроллить список вверх, потом вниз к последним сообщениям.
- **Ожидание:** Минимум ререндеров Chat/MessagesList; renderItem только для новых ячеек, попавших в viewport. MessageItem не должен сыпаться на каждое движение скролла.
- **Пришли:** Логи за время одного цикла «проскроллил вверх — проскроллил вниз».

---

## 3. Long press по одному сообщению
- **Действие:** Долгое нажатие на **одно** сообщение (открытие контекстного меню).
- **Ожидание:** Либо 0 ререндеров MessageItem, либо ререндер только этого сообщения (если меню как-то его затрагивает). Остальные видимые сообщения не должны ререндериться.
- **Пришли:** Логи с момента long press до появления меню.

---

## 4. Режим выбора (selection mode)
- **Действие:** Включить режим выбора сообщений (как у тебя включается — long press / кнопка и т.п.), **не** выбирать сообщения галочками.
- **Ожидание:** Один ререндер Chat (меняется isSelectionMode). MessagesList может ререндернуться. MessageItem — только у видимых ячеек один раз (им передаётся isSelectionMode), без повторных ререндеров.
- **Пришли:** Логи с момента входа в режим выбора.

---

## 5. Выбор одного сообщения галочкой
- **Действие:** В режиме выбора нажать галочку на **одном** сообщении.
- **Ожидание:** Ререндер Chat (обновился selectedMessages). Ререндер только **одного** MessageItem (у которого изменился isSelected). Остальные видимые сообщения не ререндерятся.
- **Пришли:** Логи с момента нажатия галочки.

---

## 6. Снять выбор с сообщения
- **Действие:** Снять галочку с выбранного сообщения.
- **Ожидание:** Аналогично п.5 — ререндер только одного MessageItem.
- **Пришли:** Логи.

---

## 7. Отправка нового сообщения
- **Действие:** Написать и отправить одно новое сообщение в чат.
- **Ожидание:** Реакция на обновление данных (Chat, возможно MessagesList, processMessages). renderItem для нового сообщения и, при необходимости, соседних. Без массового ререндера всех видимых MessageItem из-за store.
- **Пришли:** Логи с момента отправки до появления сообщения в списке.

---

## Как присылать отчёт
Для каждого пункта скопируй все строки с `[ChatRender]` и приложи к номеру пункта (можно просто: «П.3: [вставь логи]»). В конце можно добавить итог: сколько раз за сценарий отрендерились Chat, MessagesList, MessageItem, renderItem — так можно будет сравнить до/после следующих оптимизаций.
